# Challenge : Custom `find` Implementation

## Business Requirement
Build a Java program that searches for files and directories matching criteria like the Unix `find` command.

## Program Interface
```bash
java FIND [path...] [expression]
```

## Business Requirements

### Requirement 1: Starting directory
Search starting from current directory if no path specified, or from specified directory.

### Requirement 2: Name matching (-name pattern)
Find files/directories whose name matches the given pattern (supports wildcards: `*` and `?`).

### Requirement 3: Type filtering (-type type)
Filter by type:
- `f`: regular files only
- `d`: directories only
- `l`: symbolic links only

### Requirement 4: Size filtering (-size [+-]n[cwbkMG])
Find files by size:
- `-size 100c`: exactly 100 bytes
- `-size +10k`: larger than 10 kilobytes
- `-size -5M`: smaller than 5 megabytes
- Suffixes: `c` bytes, `w` 2-byte words, `b` 512-byte blocks, `k` KB, `M` MB, `G` GB

### Requirement 5: Time filtering (-mtime, -atime, -ctime)
- `-mtime n`: modified exactly n days ago
- `-mtime +n`: modified more than n days ago
- `-mtime -n`: modified less than n days ago
- Support `-atime` (access time) and `-ctime` (change time) similarly

### Requirement 6: User/group filtering (-user name, -group name)
Find files owned by specific user or group.

### Requirement 7: Permission filtering (-perm mode)
Find files with specific permissions:
- `-perm 755`: exact match
- `-perm -755`: at least these permissions (all bits set)
- `-perm /755`: any of these permissions (any bit set)

### Requirement 8: Execute command (-exec command {} \;)
For each found file, execute a command, replacing `{}` with the file path.

### Requirement 9: Print results (-print)
Default action (print path of each found file). Also support `-print0` for null-separated output.

### Requirement 10: Logical operators (-and, -or, -not)
Combine expressions:
- `-and` (default between tests)
- `-or`
- `-not` or `!`

### Requirement 11: Max depth (-maxdepth n)
Limit search to specified directory depth.

### Requirement 12: Follow symlinks (-follow)
When specified, follow symbolic links instead of treating them as files.

## Acceptance Tests

# Test 1: Find by name
java FIND . -name "*.java"
Expected: All .java files in current directory and subdirectories

# Test 2: Find directories only
java FIND . -type d -name "src"
Expected: All directories named "src"

# Test 3: Find by size
java FIND . -size +1M -type f
Expected: All files larger than 1 megabyte

# Test 4: Find by modification time
java FIND . -mtime -7 -name "*.log"
Expected: Log files modified in last 7 days

# Test 5: Find by user
java FIND /home -user "john"
Expected: All files owned by user "john"

# Test 6: Find by permission
java FIND . -perm -755 -type f
Expected: Files with rwxr-xr-x (owner can write, others can read/execute)

# Test 7: Execute command on found files
java FIND . -name "*.tmp" -exec rm {} \;
Expected: Deletes all .tmp files

# Test 8: Logical OR
java FIND . -name "*.java" -or -name "*.xml"
Expected: All Java and XML files

# Test 9: Logical NOT
java FIND . -not -name "*.class" -type f
Expected: All files that are NOT .class files

# Test 10: Max depth
java FIND . -maxdepth 2 -name "*.txt"
Expected: Only finds .txt files up to 2 levels deep

# Test 11: Multiple starting paths
java FIND /tmp /var/log -name "*.log"
Expected: Searches both directories

# Test 12: Complex expression
java FIND . -type f -name "*.java" -and -size +10k -and -not -name "*Test*"
Expected: Java files >10KB that are not test files

# Test 13: Print0 for handling filenames with spaces
java FIND . -name "*.pdf" -print0 | xargs -0 ls -l
Expected: Handles filenames with spaces correctly

# Test 14: No matches
java FIND . -name "nonexistent*.xyz"
Expected: No output

# Test 15: Permission denied
java FIND /root -name "*.conf"
Expected: Error: Permission denied for /root, continues other paths

# Test 16: Symlink handling
java FIND . -follow -name "*.so"
Expected: Follows symlinks to find .so files

# Test 17: Empty directories
java FIND . -type d -empty
Expected: All empty directories

# Test 18: Regex name matching (bonus)
java FIND . -regex ".*[0-9]+\.java"
Expected: Java files with numbers in name
```
