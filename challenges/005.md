
# Challenge 5: Event Bus with Priority and Filtering

## Business Requirement
Build a publish-subscribe event bus where subscribers can receive events based on type, priority, and custom filters.

## Business Requirements

### Requirement 1: Event definition
Events must have:
- Type (String, e.g., "USER_LOGIN", "ORDER_PLACED")
- Timestamp (when event occurred)
- Payload (generic type, any object)
- Priority (HIGH, MEDIUM, LOW)

### Requirement 2: Subscribe by event type
Subscribers can register to receive events of specific types only.

### Requirement 3: Priority-based delivery
Events with higher priority must be delivered before lower priority events, regardless of publication order.

### Requirement 4: Custom filter support
Subscribers can provide a predicate (filter) to receive only events matching custom conditions (e.g., only orders over $100).

### Requirement 5: Async delivery (optional)
Events should be delivered asynchronously so publishers aren't blocked by slow subscribers.

### Requirement 6: Subscriber management
Support:
- `subscribe(eventType, subscriber, priority, filter)`
- `unsubscribe(subscriber)` - remove from all event types
- `unsubscribe(eventType, subscriber)` - remove from specific type

### Requirement 7: Event publishing
`publish(event)` must:
- Find all subscribers for event type
- Apply filters
- Deliver events in priority order

### Requirement 8: Error handling
If a subscriber throws an exception during event processing:
- Log the error
- Do not affect other subscribers
- Optionally send to dead letter queue

### Requirement 9: Event history
Maintain history of last 100 events published (for debugging).

### Requirement 10: Wildcard subscriptions
Support subscribing to all event types using wildcard "*".
