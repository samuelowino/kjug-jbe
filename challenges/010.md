# Challenge : Custom `tail` Implementation

## Business Requirement
Build a Java program that displays the last part of files like the Unix `tail` command.

## Program Interface
```bash
java TAIL [options] [file...]
```

## Business Requirements

### Requirement 1: Display last 10 lines
When run without options, display the last 10 lines of each file.

### Requirement 2: Multiple files
When multiple files are provided, display a header before each file's content:
```
==> file1.txt <==
[content]

==> file2.txt <==
[content]
```

### Requirement 3: Number of lines (-n N)
When `-n N` is provided, display last N lines instead of default 10. Support `-n +N` to start from line N instead of end.

### Requirement 4: Number of bytes (-c N)
When `-c N` is provided, display last N bytes of each file. Support `-c +N` to start from byte N.

### Requirement 5: Follow mode (-f flag)
When `-f` flag is provided, follow the file as it grows (like `tail -f`). Keep reading new lines appended to the file until interrupted.

### Requirement 6: Follow by name (-F flag)
When `-F` flag is provided, follow file even if it's rotated (renamed or recreated). Detect when file is replaced and continue following the new file.

### Requirement 7: Quiet mode (-q flag)
When `-q` flag is provided, never print headers even when multiple files.

### Requirement 8: Verbose mode (-v flag)
When `-v` flag is provided, always print headers even when only one file.

### Requirement 9: Read from stdin
If no files are provided, read from standard input (System.in).

### Requirement 10: Efficient large file handling
For large files, seek to near the end rather than reading the entire file.

### Requirement 11: Binary file detection
Detect binary files and optionally show a warning.

### Requirement 12: Sleep interval in follow mode (-s N)
When following, wait N seconds between checks (default 1.0 seconds).

## Acceptance Tests

```bash
# Test 1: Last 10 lines default
java TAIL app.log
Expected: Shows last 10 lines of app.log

# Test 2: Custom number of lines
java TAIL -n 20 app.log
Expected: Shows last 20 lines

# Test 3: Start from line N
java TAIL -n +5 app.log
Expected: Shows from line 5 to end of file

# Test 4: Last N bytes
java TAIL -c 100 app.log
Expected: Shows last 100 bytes

# Test 5: Start from byte N
java TAIL -c +50 app.log
Expected: Shows from byte 50 to end

# Test 6: Multiple files
java TAIL file1.txt file2.txt
Expected: 
==> file1.txt <==
[last 10 lines of file1]

==> file2.txt <==
[last 10 lines of file2]

# Test 7: Follow mode
java TAIL -f app.log
Expected: Shows last 10 lines, then waits for new lines, printing them as they appear
Press Ctrl+C to exit

# Test 8: Follow with rotation
java TAIL -F app.log
# Simulate log rotation: mv app.log app.log.old; echo "new" > app.log
Expected: Detects rotation, continues following new app.log

# Test 9: Quiet mode
java TAIL -q file1.txt file2.txt
Expected: No headers, just concatenated content

# Test 10: Verbose mode with single file
java TAIL -v app.log
Expected: Shows "==> app.log <==" header even with one file

# Test 11: Read from stdin
cat app.log | java TAIL -n 5
Expected: Shows last 5 lines from stdin

# Test 12: Large file efficiency
java TAIL -n 1000 hugefile.log (1GB file)
Expected: Completes quickly without reading entire file

# Test 13: Non-existent file
java TAIL nonexistent.txt
Expected: Error: nonexistent.txt: No such file or directory

# Test 14: Empty file
java TAIL empty.txt
Expected: No output

# Test 15: Follow with custom interval
java TAIL -f -s 0.5 app.log
Expected: Checks for new lines every 0.5 seconds

# Test 16: Lines vs bytes combination (should error)
java TAIL -n 10 -c 100 app.log
Expected: Error: Cannot use -n and -c together

# Test 17: Zero lines
java TAIL -n 0 app.log
Expected: No output

# Test 18: More lines than file
java TAIL -n 99999 smallfile.txt
Expected: Shows entire file

# Test 19: Follow mode with multiple files
java TAIL -f file1.log file2.log
Expected: Shows headers when content appears from each file

# Test 20: Binary file warning
java TAIL binary.dat
Expected: Warning: binary.dat appears to be a binary file, output may be garbled
```

## Implementation Checklist
- [ ] Parse command line arguments
- [ ] Implement line counting from end (RandomAccessFile seek)
- [ ] Implement byte counting from end
- [ ] Implement line start position calculation
- [ ] Implement follow mode with FileReader and watch for changes
- [ ] Implement rotation detection (track inode or file length decreasing)
- [ ] Implement multi-file handling with headers
- [ ] Handle stdin reading
- [ ] Detect binary files (check for null bytes or non-printable chars)
- [ ] Implement sleep interval
- [ ] Handle graceful shutdown (Ctrl+C)
- [ ] Test all 20 cases
