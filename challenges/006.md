# Challenge 3: Custom Generic Cache with Eviction

## Business Requirement
Build a generic in-memory cache that automatically removes oldest entries when it reaches maximum size.

## Business Requirements

### Requirement 1: Generic type support
The cache must work with any key and value types (generics).

### Requirement 2: Size limitation
The cache must have a maximum capacity (configurable at construction, default 100 entries).

### Requirement 3: LRU eviction policy
When cache exceeds maximum capacity, the least recently used entry must be removed automatically.

### Requirement 4: Core operations
Support:
- `put(key, value)` - add or update entry, marks as most recently used
- `get(key)` - retrieve value, marks key as most recently used, returns null if not found
- `remove(key)` - delete specific entry
- `clear()` - remove all entries
- `size()` - return current number of entries
- `containsKey(key)` - check if key exists

### Requirement 5: Access order tracking
The cache must track access order (both put and get operations count as "use").

### Requirement 6: Thread safety
All operations must be safe for concurrent access from multiple threads.

### Requirement 7: Null handling
- Null keys are not allowed (throw IllegalArgumentException)
- Null values are allowed (can be stored and retrieved)

### Requirement 8: Statistics
The cache must provide:
- Hit count (number of successful gets)
- Miss count (number of gets where key not found)
- Eviction count (number of entries removed due to size limit)
- Current hit ratio (hits / total gets)

### Requirement 9: Peek operation
Provide a `peek(key)` method that retrieves value WITHOUT marking as recently used (doesn't affect LRU order).

### Requirement 10: Entry iteration
Provide a way to get all current keys or values (snapshot, not live view).

## Acceptance Tests

```bash
# Test 1: Basic put and get
Cache<String, Integer> cache = new Cache<>(3);
cache.put("A", 1);
cache.get("A")
Expected: 1

# Test 2: LRU eviction
Cache<String, Integer> cache = new Cache<>(3);
cache.put("A", 1); // A
cache.put("B", 2); // B, A
cache.put("C", 3); // C, B, A
cache.get("A");    // A becomes most recent: A, C, B
cache.put("D", 4); // D added, B evicted (least recent)
cache.get("B")
Expected: null (B was evicted)
cache.containsKey("A")
Expected: true
cache.containsKey("C")
Expected: true
cache.containsKey("D")
Expected: true

# Test 3: Update existing key
Cache<String, Integer> cache = new Cache<>(3);
cache.put("A", 1);
cache.put("A", 2);
cache.get("A")
Expected: 2

# Test 4: Null key rejection
Cache<String, Integer> cache = new Cache<>(3);
cache.put(null, 1)
Expected: IllegalArgumentException

# Test 5: Null value allowed
Cache<String, Integer> cache = new Cache<>(3);
cache.put("A", null);
cache.get("A")
Expected: null (but key exists)
cache.containsKey("A")
Expected: true

# Test 6: Thread safety
10 threads each put 100 entries, 10 threads each get 100 entries
Expected: No exceptions, cache size never exceeds limit, all operations complete

# Test 7: Statistics tracking
Cache<String, Integer> cache = new Cache<>(3);
cache.put("A", 1);
cache.put("B", 2);
cache.get("A"); // hit
cache.get("C"); // miss
cache.get("A"); // hit
cache.get("B"); // hit
cache.getHitCount()
Expected: 3
cache.getMissCount()
Expected: 1
cache.getHitRatio()
Expected: 0.75 (3/4)

# Test 8: Eviction counting
Cache<String, Integer> cache = new Cache<>(2);
cache.put("A", 1);
cache.put("B", 2);
cache.put("C", 3); // evicts A
cache.put("D", 4); // evicts B
cache.getEvictionCount()
Expected: 2

# Test 9: Peek operation
Cache<String, Integer> cache = new Cache<>(2);
cache.put("A", 1);
cache.put("B", 2);
cache.peek("A"); // doesn't affect order
cache.put("C", 3); // should evict B (least recent), not A
cache.containsKey("A")
Expected: true
cache.containsKey("B")
Expected: false

# Test 10: Snapshot iteration
Cache<String, Integer> cache = new Cache<>(3);
cache.put("A", 1);
cache.put("B", 2);
cache.put("C", 3);
Set<String> keys = cache.keySet(); // snapshot
cache.put("D", 4); // modifies original
keys.size()
Expected: 3 (snapshot unaffected by later changes)
