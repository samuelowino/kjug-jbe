# Challenge : Custom `grep` Implementation

## Business Requirement
Build a Java program that searches for patterns in files like the Unix `grep` command.

## Program Interface
```bash
java GREP [options] pattern [file...]
```

## Business Requirements

### Requirement 1: Basic pattern search
Search for a literal string pattern in one or more files. Print each line that contains the pattern.

### Requirement 2: Multiple files support
When multiple files are provided, print filename before each matching line.

### Requirement 3: Read from standard input
If no files are provided, read from standard input (System.in).

### Requirement 4: Line numbers (-n flag)
When `-n` flag is provided, prefix each matching line with its line number (starting at 1).

### Requirement 5: Case-insensitive search (-i flag)
When `-i` flag is provided, ignore case differences when matching.

### Requirement 6: Invert match (-v flag)
When `-v` flag is provided, print lines that do NOT match the pattern.

### Requirement 7: Count matches (-c flag)
When `-c` flag is provided, print only the count of matching lines per file, not the lines themselves.

### Requirement 8: Recursive search (-r flag)
When `-r` flag is provided, recursively search all files in directories (not just direct files).

### Requirement 9: Whole word matching (-w flag)
When `-w` flag is provided, match only whole words (pattern surrounded by word boundaries).

### Requirement 10: Show only filename (-l flag)
When `-l` flag is provided, print only the names of files that contain at least one match.

### Requirement 11: Context lines (-A, -B, -C flags)
- `-A num`: Show `num` lines after each match
- `-B num`: Show `num` lines before each match
- `-C num`: Show `num` lines both before and after each match

### Requirement 12: Regular expression support (bonus)
Support basic regex: `.` (any char), `*` (zero or more), `+` (one or more), `?` (zero or one), `[abc]` (character class).

## Acceptance Tests

# Test 1: Basic search single file
java GREP "error" app.log
Expected: All lines containing "error"

# Test 2: Search multiple files
java GREP "TODO" *.java
Expected: 
File1.java:    // TODO: fix this
File2.java:    // TODO: implement

# Test 3: Line numbers
java GREP -n "error" app.log
Expected: "15: Error: connection failed"

# Test 4: Case insensitive
java GREP -i "error" app.log
Expected: Matches "Error", "ERROR", "error"

# Test 5: Invert match
java GREP -v "debug" app.log
Expected: All lines NOT containing "debug"

# Test 6: Count matches
java GREP -c "error" *.log
Expected: 
app1.log: 3
app2.log: 0
app3.log: 7

# Test 7: Recursive search
java GREP -r "class" .
Expected: Searches all .java files in all subdirectories

# Test 8: Whole word matching
java GREP -w "to" file.txt
Expected: Matches "to" but not "today" or "into"

# Test 9: Show only filenames with matches
java GREP -l "error" *.log
Expected: 
app1.log
app3.log

# Test 10: Context lines after match
java GREP -A 2 "Exception" app.log
Expected: Shows matching line + 2 lines after each match

# Test 11: Context lines before match
java GREP -B 2 "Exception" app.log
Expected: Shows 2 lines before + matching line

# Test 12: Context lines around match
java GREP -C 1 "Exception" app.log
Expected: Shows 1 line before, matching line, 1 line after

# Test 13: Read from stdin
cat file.txt | java GREP "pattern"
Expected: Searches stdin content

# Test 14: No matches
java GREP "xyz" file.txt
Expected: No output, exit code 1

# Test 15: File not found
java GREP "pattern" nonexistent.txt
Expected: Error: nonexistent.txt: No such file or directory
Continue processing other files

# Test 16: Binary file handling
java GREP "text" binary.dat
Expected: Detect binary file, optionally skip or show message
```

## Implementation Checklist
- [ ] Parse command line arguments and flags
- [ ] Read files line by line (handle large files efficiently)
- [ ] Implement pattern matching (String.contains or regex)
- [ ] Implement case-insensitive matching
- [ ] Implement invert matching
- [ ] Implement line number tracking
- [ ] Implement recursive directory walking
- [ ] Implement word boundary detection
- [ ] Implement context line buffering
- [ ] Handle stdin reading
- [ ] Set appropriate exit codes (0=match found, 1=no match, 2=error)
- [ ] Test all 16 cases
