## Java ls commad

## Business Requirement
Build a Java program that lists directory contents like the Unix `ls` command, supporting common options.

## Program Interface
```bash
java LS [options] [directory]
```

## Business Requirements

### Requirement 1: List directory contents
When run without arguments, the program must list all files and directories in the current working directory.

### Requirement 2: Support target directory
Accept a directory path as argument and list contents of that directory instead of current directory.

### Requirement 3: Hidden files exclusion (-a flag)
By default, hidden files (starting with `.` on Unix) must NOT be shown. When `-a` flag is provided, show all files including hidden ones.

### Requirement 4: Long format (-l flag)
When `-l` flag is provided, show detailed information:
- File permissions (read/write/execute for owner/group/others)
- Number of hard links
- Owner name
- Group name
- File size in bytes
- Last modification timestamp
- File name

### Requirement 5: Human-readable sizes (-h flag)
When combined with `-l`, show file sizes in human-readable format (B, K, M, G) instead of raw bytes.

### Requirement 6: Reverse order (-r flag)
When `-r` flag is provided, list files in reverse alphabetical order.

### Requirement 7: Sort by modification time (-t flag)
When `-t` flag is provided, sort files by last modified time (newest first) instead of alphabetically.

### Requirement 8: Recursive listing (-R flag)
When `-R` flag is provided, recursively list all subdirectories and their contents.

### Requirement 9: Column output
Default output (without `-l`) must format filenames in columns, fitting as many as possible in the terminal width.

### Requirement 10: Error handling
- If directory does not exist → print error message to stderr and exit with non-zero code
- If path is a file → print just that file's information
- If permission denied → print error message and continue with accessible files

### Requirement 11: Multiple flags combination
Support combining flags (e.g., `-la`, `-l -a`, `-al` all mean the same thing).

### Requirement 12: Special entries handling
- Current directory `.` must NOT be shown in listings
- Parent directory `..` must NOT be shown in listings (unless `-a` is used)

## Acceptance Tests

# Test 1: List current directory
java LS
Expected: Column-formatted list of files (excluding hidden files)

# Test 2: List specific directory
java LS /tmp
Expected: Contents of /tmp directory

# Test 3: Show hidden files
java LS -a
Expected: Includes files starting with '.'

# Test 4: Long format
java LS -l
Expected: 
drwxr-xr-x 2 user group 4096 2026-04-21 10:30 Documents
-rw-r--r-- 1 user group 1024 2026-04-20 15:45 file.txt

# Test 5: Human-readable sizes
java LS -lh
Expected: File sizes shown as "1.2K", "4.5M", "2.1G" instead of bytes

# Test 6: Reverse order
java LS -r
Expected: Files listed in reverse alphabetical order

# Test 7: Sort by time (newest first)
java LS -t
Expected: Most recently modified files at top

# Test 8: Recursive listing
java LS -R
Expected:
.:
file1.txt  dir1

./dir1:
file2.txt  file3.txt

# Test 9: Non-existent directory
java LS /nonexistent
Expected: Error: /nonexistent: No such file or directory
Exit code: non-zero

# Test 10: File instead of directory
java LS /etc/passwd
Expected: Shows just /etc/passwd file info

# Test 11: Combined flags
java LS -la
Expected: Long format with hidden files included

# Test 12: Permission denied
java LS /root
Expected: Error: /root: Permission denied (if no permission)
Other accessible files still listed
```
