# Spring Boot Developer Assessment
Implement a secure session management REST API using Spring Boot with JPA, Redis, and Spring Security.

---

## Task 1: Entity Layer ✅

Create the `User` entity class with the following specifications:

- Table name: `users`
- Fields:
  - `id`: Long, auto-generated primary key
  - `username`: String, unique, not null, length min 3 max 50
  - `password`: String, not null, min length 6
  - `email`: String, unique, not null, valid email format
  - `createdAt`: LocalDateTime, automatically set on creation
- Create `UserRepository` interface extending `JpaRepository`

---

## Task 2: REST Endpoints

Implement the following endpoints in `AuthController`:

| Method | Endpoint | Request | Response | Status Codes |
|--------|----------|---------|----------|--------------|
| POST | `/api/register` | `{"username":"string","password":"string","email":"string"}` | `{"id":long,"username":"string","email":"string"}` | 201, 400, 409 |
| POST | `/api/login` | `{"username":"string","password":"string"}` | `{"sessionId":"string"}` | 200, 401 |
| POST | `/api/logout` | (headers: `Authorization: Bearer {sessionId}`) | `{"message":"Logged out"}` | 200, 401 |
| GET | `/api/sessions` | (headers: `Authorization: Bearer {sessionId}`) | `[{"sessionId":"string","createdAt":"ISO datetime"}]` | 200, 401 |
| DELETE | `/api/sessions/{sessionId}` | (headers: `Authorization: Bearer {sessionId}`) | `{"message":"Session revoked"}` | 200, 401, 404 |

---

## Task 3: Redis Session Storage

Implement `SessionService` with:

- On login: Generate random UUID sessionId, store in Redis with key `session:{sessionId}` → `userId`, TTL 30 minutes
- Store user's sessions index: `user:{userId}:sessions` as Redis Set of sessionIds
- On logout/session revocation: Delete from both primary key and user's sessions set
- On `/api/sessions`: Retrieve all sessionIds from user's sessions set, fetch each session's metadata, return sorted by createdAt

---

## Task 4: Spring Security

Implement security configuration:

- Permit all: `/api/register`, `/api/login`
- Secure all other `/api/**` endpoints
- Create `SessionAuthenticationFilter` extending `OncePerRequestFilter` that:
  - Extracts Bearer token from Authorization header
  - Queries Redis `session:{sessionId}` to get userId
  - If exists, sets `Authentication` in `SecurityContextHolder` with username = userId
  - If not exists or missing header, returns 401 Unauthorized

---

## Task 5: Error Handling

Implement global exception handling for:

- `MethodArgumentNotValidException` → 400 Bad Request
- `DuplicateKeyException` (username/email conflict) → 409 Conflict
- `SessionNotFoundException` → 404 Not Found
- `AuthenticationException` → 401 Unauthorized
